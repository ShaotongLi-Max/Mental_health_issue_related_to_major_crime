The reason the \tag{} syntax doesn't work for cross-referencing equations in Quarto is that \tag{} is used in LaTeX specifically to customize the numbering of equations, but it does not create a label that can be cross-referenced. Quarto (and Pandoc, which Quarto is built on) requires the use of \label{} to assign cross-reference labels.

Differences:
\tag{}: In LaTeX, \tag{} is used to manually specify the number or tag that appears next to the equation, but it doesn't create a cross-reference target.

\label{}: This is used in LaTeX and Quarto for assigning a label to the equation, which can then be used for cross-referencing with @eq-label.

Why \label{} Works in Quarto:
Quarto relies on Pandoc's internal referencing system, which uses \label{} to generate internal cross-reference targets for figures, tables, and equations. By using \label{}, Quarto knows that it needs to track that equation for cross-referencing.

Example Comparison:
\tag{}:

markdown
Â§çÂà∂‰ª£Á†Å
$$

$$
This will display the equation with the number 1, but you cannot cross-reference it in Quarto.

\label{}:

Â§çÂà∂‰ª£Á†Å
$$

$$
This allows you to reference it as @eq-energy.

Conclusion:
If you want to cross-reference equations in Quarto, you need to use \label{} instead of \tag{} because Quarto recognizes \label{} for internal linking and citation, while \tag{} is used purely for display purposes and does not generate a cross-reference target.
